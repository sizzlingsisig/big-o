# Big O: Technical Debt

A reverse-growth arcade game where you play as a bloated, inefficient algorithm ($O(2^n)$) that must "eat" refactor packets to optimize into a god-like $O(1)$ state before Technical Debt crashes the CPU.

## The Mechanical Twist

In most arcade games, you grow bigger and stronger. In **Big O: Technical Debt**, you start as a massive, sluggish, laggy mess and must shed "bloat" to become fast and efficient. Your Complexity Tier directly controls:

- **Speed**: From 100 px/s (Exponential) to 220 px/s (Constant)
- **Collider Size**: From 96px to 22px
- **Inertia**: From 0.9 to 0.0
- **Input Lag**: From 0.3s to 0.0s

Collect **Clean Code Packets** to lower your Big O tier, while avoiding Technical Debt enemies that increase your RAM usage.

---

## Controls

| Input | Action |
|-------|--------|
| Mouse Move | Algorithm drifts toward cursor (inertia/lag depends on tier) |
| Spacebar | Activate "Zombie Fork" - clears 50% RAM but drops complexity tier |
| Escape | Pause / Quit (on game over) |
| Up Arrow (Debug) | Force refactor (dev builds) |
| Down Arrow (Debug) | Force accumulate debt (dev builds) |

## Building & Running

### Prerequisites
- Godot 4.6+

### Running the Game
1. Open `project.godot` in Godot 4.6+
2. Press F5 to run the main scene (`scenes/core/main_screen.tscn`)

### Exporting
1. Project > Export
2. Select target platform (PC/Web)
3. Click Export Project
